# Нові групи товарів і правила фіду

## Що додано в логіку

Проєкт більше не вважає всі товари спінінгами. Логіка групування та фідів тепер розрізняє:

- `Спінінги`
- `Вудки`
- `Волосінь`
- `Флюрокарбон`
- `Шок-лідер`
- `Повідці`
- `Бойли`
- `Поп-ап насадки`
- `Зернові`
- `Пелетс та гранула`
- `Мікси та стік-мікси`
- `Ліквіди`
- `Кивки`
- `Сигналізатори`
- `Аксесуари для підставки`

## Які характеристики закладаємо у фід

### Вудки / спінінги

- Базові поля: `Тип вудилища`, `Бренд`
- Варіантні поля: `Довжина`, `Тест`, `Стрій`
- Додатково бажано мати вручну: `Матеріал бланка`, `Кількість секцій`, `Транспортна довжина`, `Вага`

### Волосінь / флюрокарбон / шок-лідер

- Базові поля: `Тип`, `Матеріал`, `Призначення`
- Варіантні поля: `Діаметр`, `Довжина`, `Розривне навантаження`, `PE`
- Для шок-лідера окремо бажано: `Тип конуса`, `Колір`, `Кількість відрізків`

### Повідці

- Базові поля: `Тип`, `Матеріал`, `Призначення`
- Варіантні поля: `Довжина`, `Діаметр`, `Розривне навантаження`, `Розмір`
- Для готових коропових повідців бажано вручну: `Тип гачка`, `Кількість у пачці`

### Бойли / поп-ап / зернові / пелетс / мікси / ліквіди

- Базові поля: `Тип насадки` або `Тип атрактанту`, `Призначення`, `Аромат`
- Варіантні поля: `Діаметр`, `Вага`, `Об'єм`, `Кількість в упаковці`
- Для поп-апів додатково: `Плавучість`
- Для ліквідів додатково: `Сумісність з ПВА`, `Рекомендоване дозування`

### Кивки

- Базові поля: `Тип`, `Матеріал`, `Сезон`
- Варіантні поля: `Тест`, `Довжина`, `Розмір`
- За можливості вручну: `Товщина`

### Сигналізатори / свінгери / світлячки

- Базові поля: `Тип`, `Підтип`
- Варіантні поля: `Довжина`, `Діаметр`, `Кількість в упаковці`
- Для електронних сигналізаторів бажано вручну: `Живлення`, `Тип індикації`, `Чутливість`, `Дальність`

## Імпорт з Excel

Новий імпортер `src/import_xlsx_to_ukrsklad.py` читає xlsx-файли з Telegram Desktop, визначає цільову групу товарів і може:

- створити відсутні групи `TIP`
- додати або оновити товар у `TOVAR_NAME`
- оновити залишок у `TOVAR_ZAL`
- зберегти preview у `data/xlsx_import_preview.json`

За замовчуванням імпортер працює у dry-run, щоб можна було спочатку перевірити результат.
